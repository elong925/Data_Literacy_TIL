# 1️⃣ 개념정리

## 01. 자꾸만 늘어가는 '방법맨'이란

```
✅ 학습 목표 :
* '방법맨'이란 무엇인지 설명할 수 있다.
* 문제 해결보다 기법 자체에 집착하게 되는 사고 방식의 위험성을 설명할 수 있다. 
```

> <문제 해결 프로세스>
>> 문제 -> 원인 -> 해결 방안
>> 세 요소는 각각 독립적, 순서가 정해져있음
>> 분석할 때 항상 어떤 단계에 있는지 생각하며 진행하기!

### 문제 해결 프로세스에서 흔히 하는 실수
- 해결 방안에 무턱대고 달려든다. 
    - 주제를 선정하여 데이터를 수집하고 현상을 파악하는 것이 일반적. 
    - 시각화 결과를 바탕으로 바로 해결방안을 강구하려고 함. 
    - 과연 결론이 정말 **데이터 기반**일까? 
    - **데이터에서 확인한 내용이 과연 결론을 입증할 수 있는 정보라고 할 수 있을까?**  -> ❌
- '방법맨' : 철저히 데이터에 기반하여 결론을 내리는 것이 아닌, 무작정 해결방안을 생각하려는 작업자

### 무턱대고 '해결 방안'으로 뛰어들기 전에 해야 하는 행동
- '원인'을 분석한다. 
    - 전제 : '문제'가 올바르게 정의되어야 한다. 
    - 해결방안은 문제에 적용하는 것이 아닌, 문제를 일으킨 '원인'에 적용하는 것이기 때문!
- 다음 질문을 던져보자.
    - 어째서 (다른 수단이 아닌) 그것이 필요한지?
    - 그로 인해 어떤 것을 실현 또는 해결 가능한지? 그리고 효과적인 결과가 나올 것이라는 근거는 어디에 있는지?




## 02. 방법맨이 돼버리는 구조

```
✅ 학습 목표 :
* 방법맨이 되어버리는 구조적 요인을 이해한다.
* 분석 기법보다 '문제 인식 -> 사고 구조 설계'가 우선임을 인식한다. 
```

> '생각한다'는 말의 의미
>> 풍부한 창의성으로 혁신적인 아이디어를 생각해낸다.
>> 논리적으로 꾸준히 사물이나 현상의 원리를 추구한다. 

💡"구체적으로 머리 써서 생각해봐" <br>
-> 기발한 아이디어를 떠올리려고 노력함 -> 다른 팀과 비교하며 아이디어를 평가함 <br>
-> 위는 문제 해결 프로세스를 고려하지 못한 것임 <br>
=> 분석의 목적과 문제, 원인을 확실히 정의한 후에 해결방안을 생각해야 함!





## 03. '왜(원인)' 그런지 끈질기게 생각하라

```
✅ 학습 목표 :
* 기법을 적용하기 전에 '무엇을, 왜 분석하는가'를 지속적으로 질문하는 태도를 기른다.
* 단순 결과에 안주하지 않고 원인을 집요하게 파고드는 분석 사고력을 기른다. 
```

### '방법맨'이 되는 것을 피하려면?
1. 문제 해결 프로세스를 항상 염두에 둔다.
2. 내가 어느 단계에 있는지 파악한다. 

### '원인'의 구조는 단순하지 않다. 
- 원인 후보를 추출한 것과 원인을 정의한 것은 다르다. 
- "어째서", "왜" 라는 질문을 반복해서 **본질적인 원인**까지 파고들어야 한다. 
    - 얼마나 "왜"를 반복하면 돼?
    - 두 지표 사이에 관련성이 있다고 판명되면! 최종단계에 들어가서 '해결방안'을 제시하는 것. 

> 해결방안을 고민하는 것은 마지막 단계임을 명심해라!

# 2️⃣ 확인 문제

## 문제

> **🧚Q. 다트비 학회에서는 최근 몇 주간 학회원들의 세션 참여율이 눈에 띄게 감소하고 있었다. 이에 총괄팀장인 권택이는 '그냥 요즘 다들 바쁜건가'라고 생각을 했고 이에 대한 의견을 공유했을 때 병욱이는 '발표 주제가 너무 어려워서 그런 것 같아' 라고 말을 했다. 결국 이 둘은 같이 문제를 정식으로 분석해보기로 했다.**
>
> **위 사례에서 다음 세 가지의 과업을 순서대로 수행해보세요.**
>
> **1. 문제 정의 : '참여율이 낮아졌다'는 현상을 데이터 관점에서 정확하게 정의해보세요.**
>
> **2. 원인 열거 : 참여율 감소의 가능한 원인들을 최소 3가지 이상 나열해보세요.**
>
> - **표면적 요인 외에도 잠재적, 구조적 원인도 포함하면 더 좋습니다.**
>
> **3. 해결 방안 도출 : 위에서 제시한 각 원인에 대해, 현실적으로 적용 가능한 대응 방안을 1:1로 매칭하여 제시해보세요.**

<!--학습한 개념을 활용하여 자유롭게 설명해 보세요. 구체적인 예시를 들어 설명하면 더욱 좋습니다.-->

```
1. 문제 정의 : 세선 참여율에 대한 정량적 정의 =  (세션 참여 인원수/전체 학회원 수)*100 , 최근 한 달간의 세션 참여율 추이를 본다 => 세션 참여율이 약 ~%감소하였다. 

2. 원인 열거 : 발표 주제의 난이도, 시험 기간과 중간과제의 기간이 겹침, 세션 내용의 흥미 부족, 운영 시스템 피드백 부재 등

3. 해결 방안 도출 

1) 발표 주제 난이도 : 발표 주제의 난이도를 단계별로 구분하여 학회원이 수준을 선택할 수 있도록 함. 발표 전 예시를 공유하며 진입 장벽을 완화한다. 

2) 시험기간과 과제 기간이 겹침 : 학교의 주요 시험 및 과제 일정을 반영하여 세션 일정을 재조정하거나 비대면으로 진행한다.

3) 세션 내용의 흥미 부족 : 우수학회원 선정 제도 등의 인센티브 제도를 도입한다.

4) 운영 시스템 피드백 부재 : 피드백 설문을 세션이 끝날 때마다 배포하고 그 결과를 반영한다. 세션의 참여율, 피드백 요약 등의 내용을 주기적으로 학회원들에게 공유하여 영향력을 보여준다. 
```
